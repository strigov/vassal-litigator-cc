# prepare-hearing -- Подготовка к заседанию

Когда пользователь запускает эту команду с "$ARGUMENTS":

1. Прочитай скилл `skills/prepare-hearing/SKILL.md` и следуй его инструкциям.
2. Если `$ARGUMENTS` содержит тип документа (отзыв, ходатайство, пояснения) -- используй как вводную.
3. Если `$ARGUMENTS` содержит стратегические указания -- учти при анализе.
4. Убедись, что `.vassal/case.yaml` существует и есть хотя бы одна позиция в корне дела.
5. Загрузи полный контекст дела (case.yaml, index.yaml, позиции, отчеты о заседаниях, зеркала).
6. Делегируй анализ Opus-субагенту (model: "opus").
7. Выполни pipeline: загрузка контекста -> анализ -> preview -> генерация текста -> .docx -> обновление состояния.
