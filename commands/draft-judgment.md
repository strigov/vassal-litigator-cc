# draft-judgment -- Проект судебного акта

Когда пользователь запускает эту команду с "$ARGUMENTS":

1. Прочитай скилл `skills/draft-judgment/SKILL.md` и следуй его инструкциям.
2. Если `$ARGUMENTS` содержит указание исхода (в нашу пользу, отказ, частичное) -- используй.
3. Если `$ARGUMENTS` содержит путь к решениям судьи -- загрузи для анализа стиля.
4. Убедись, что `.vassal/case.yaml` существует и есть хотя бы одна позиция.
5. Загрузи полный контекст дела.
6. Делегируй анализ и генерацию Opus-субагенту (model: "opus").
7. Выполни pipeline: загрузка контекста -> анализ стиля судьи -> генерация проекта -> preview -> .docx -> обновление состояния.
