# Update-Index — Обновление и верификация индекса

preview→apply — preview и верификация: Claude-main (read-only), apply: Codex medium (--write).

## Предварительные условия

- Существует `.vassal/index.yaml`
- Существует `.vassal/case.yaml`

## Фаза 1 — Сканирование и Preview (Claude-main, read-only)

Claude-main не сканирует файловую систему вручную через Glob/Read. Единственный источник diff-preview — JSON от `scan_case_state.py`.

1. Выполни:
   `python3 "[PLUGIN_ROOT]/scripts/scan_case_state.py" "[CASE_ROOT]"`
2. Распарси JSON: `index_count`, `fs_count`, `new_files`, `orphans`, `stale_mirrors`.
3. Покажи Сюзерену: сколько файлов в индексе, сколько на диске, и три списка:
   - `new_files`: новые файлы на диске, отсутствующие в индексе
   - `orphans`: записи индекса, у которых нет файла на диске
   - `stale_mirrors`: зеркала, требующие пересоздания
4. Предложи действия: новые → добавить; `orphans` → удалить или обновить путь; устаревшие → пересоздать.

ЗАПРЕЩЕНО: любые записи, `cp`, `mkdir` и ручной обход файловой системы. Разрешён только read-only запуск `scan_case_state.py`.

## Фаза 2 — Подтверждение

Дождись ответа Сюзерена. Если решения уточняются или меняются — пересобери preview и покажи заново. Если действие отменено — остановись без изменений.

## Фаза 3 — Apply (Codex medium --write)

1. Сформируй `plan_body` по ответу Сюзерена: явно перечисли, какие файлы добавить (режим A), какие `orphans` как разрулить (режим B), какие зеркала пересоздать (режим C).
2. Прочитай `skills/codex-invocation/SKILL.md`.
3. Собери prompt из `prompts/file-executor-update-index.md` и подставь:
   - `case_root`
   - `plugin_root` по той же схеме, что в intake
   - `scan_scope`: `full` или явный перечень папок
   - `plan_body`: сформированный план
   - `extra_constraints`: пусто, если нет
   - `report_contract`: стандартный блок из `prompts/_preamble.md`
4. Диспатч задачу через `codex-companion.mjs` с `--write --effort medium`.
5. Мониторь выполнение.
6. Получи итоговый результат.

## Фаза 4 — Верификация (Claude-main)

1. Прочитай отчёт Codex.
2. Проверь, что изменения соответствуют согласованному плану, `.vassal/index.yaml` читается без ошибок, а в отчёте нет операций вне подтверждённого объёма.
   Для добавленных записей и пересозданных зеркал проверь, что `ocr_quality` и `ocr_quality_reason` записаны по результату `classify_ocr_quality.py`, а `needs_manual_review` соответствует правилу `ocr_quality != "ok"`.
3. Сохрани лог в `.vassal/codex-logs/{ГГГГ-ММ-ДД-ЧЧмм}-update-index.md`:
   промпт + отчёт Codex.
4. Покажи Сюзерену резюме: сколько записей добавлено, сколько `orphans` разрулено, сколько зеркал пересоздано.

## Fallback без Codex

Если Codex недоступен (`companion.mjs` отсутствует, `ECONNREFUSED`, ошибка запуска):

- Предупреди Сюзерена: `Codex недоступен, выполняю apply-фазу напрямую (медленнее, дороже по токенам)`.
- Выполни apply-фазу самостоятельно по шагам из `prompts/file-executor-update-index.md`.
- OCR и извлечение текста запускай через:
  `python3 [PLUGIN_ROOT]/scripts/extract_text.py`
- Качество OCR рассчитывай тем же скриптом, что и Codex apply:
  `python3 [PLUGIN_ROOT]/scripts/classify_ocr_quality.py`
- Лог сохрани в `.vassal/codex-logs/{дата-время}-update-index-fallback.md`.

## Идемпотентность

Повторный запуск безопасен — уже синхронизированные файлы не будут обработаны повторно.

## Граничные случаи

- Файл переименован вручную → старая запись станет `orphan`, новое имя — новым файлом. Предложи Сюзерену связать их обновлением пути.
- Пустой индекс → предложи запустить intake.
- Файл в `.vassal/raw/` появился без обработки → это нормально, `raw/` не индексируется.
