# Timeline — Хронология дела

preview→apply: Claude-main собирает контекст (read-only), Codex high строит хронологию (--write).

## Предварительные условия

- Существует `.vassal/case.yaml`
- В `.vassal/index.yaml` есть хотя бы 2 документа с датами
- Если нет — сообщи Сюзерену, что для построения хронологии нужно сначала добавить документы через intake

## Фаза 1 — Preview (Claude-main, read-only)

1. Прочитай `.vassal/case.yaml`:
   - стороны
   - предмет
   - существующий `timeline`
2. Прочитай `.vassal/index.yaml`:
   - документы с датами
   - типы документов
3. Определи, какие документы потенциально пригодны для хронологии:
   - договоры
   - переписка
   - претензии
   - процессуальные документы
   - отчёты о заседаниях из `.vassal/hearings/*.md`, если они есть
4. Составь ориентировочный план:
   - какие типы событий будут включены
   - какой примерный период покрывается
   - сколько уже есть событий в `timeline`
   - сколько из них имеют `source:user` и будут сохранены
5. Покажи Сюзерену preview:
   - источники для хронологии: `N` документов в индексе, `N` зеркал
   - существующий timeline: `N` событий, из них `K` `source:user` — будут сохранены
   - политика: `extend` или `rebuild`
   - будет создан или обновлён файл `Хронология дела.md`
   - будет обновлён `.vassal/case.yaml`

ЗАПРЕЩЕНО: любые записи, создание файлов, изменение `case.yaml`, `history.md` или `Хронология дела.md`.

## Фаза 2 — Подтверждение

Дождись ответа Сюзерена. Уточни политику: `extend` или `rebuild`.

- Если Сюзерен отменяет операцию — остановись без изменений.
- Если политика не указана явно — переспроси.
- Если Сюзерен корректирует объём работы — пересобери preview и покажи снова.

## Фаза 3 — Apply (Codex high --write)

1. Прочитай `skills/codex-invocation/SKILL.md`.
2. Собери prompt из `prompts/timeline-builder.md`, подставь:
   - `case_root`: абсолютный путь к делу
   - `plugin_root`: по 3-tier fallback из `skills/codex-invocation/SKILL.md`
   - `timeline_goal`: описание цели, например `полная хронология` или уточнение Сюзерена
   - `existing_timeline_policy`: `extend` или `rebuild`
   - `extra_constraints`: пусто, если ограничений нет
   - `report_contract`: из `prompts/_preamble.md`
3. Проверь, что все плейсхолдеры раскрыты:
   `grep -c "{{" prompt.txt`
   Результат должен быть `0`.
4. Диспатч задачу:
   `codex-companion.mjs task --background --write --effort high`
5. Мониторь выполнение по шаблону из `skills/codex-invocation/SKILL.md`.
6. Получи итоговый результат.

## Фаза 4 — Верификация (Claude-main)

1. Прочитай отчёт Codex.
2. Проверь:
   - `Хронология дела.md` создан или обновлён
   - `.vassal/case.yaml` валиден
   - события `source:user` не утеряны и не изменены
   - в отчёте нет действий вне timeline-объёма
3. Сохрани лог в `.vassal/codex-logs/{ГГГГ-ММ-ДД-ЧЧмм}-timeline.md`:
   prompt + отчёт Codex
4. Покажи Сюзерену резюме:
   - сколько событий в итоговой хронологии
   - какой период покрыт
   - путь к `Хронология дела.md`

## Фаза 5 — Предложение визуализации (опционально)

После успешной верификации спроси Сюзерена:
`Сгенерировать растровую инфографику таймлайна как sidecar-превью? (да/нет)
Инфографика — AI-generated, не встраивается в Хронология дела.md (там уже есть Mermaid-блок). Файл будет в .vassal/visuals/.`

Если ответ `да`:
- прочитай `skills/visualize/SKILL.md`
- передай `VISUAL_TYPE=timeline-infographic`, `CONTEXT=[список ключевых событий из построенного timeline: дата + краткое событие, не более 7 событий]`
- дождись результата `visualize`
- сообщи: `Инфографика таймлайна сохранена в .vassal/visuals/{ГГГГ-ММ-ДД}-timeline-infographic.png`
- правило: НЕ добавляй ссылку в `Хронология дела.md`

Если `skills/visualize/SKILL.md` ещё не существует, скажи:
`Скилл visualize недоступен (Ф5 плана v0.5.0 не применён)`

## Fallback без Codex

Если Codex недоступен:

- выполни timeline-фазу самостоятельно
- прочитай зеркала, `case.yaml`, `index.yaml` и отчёты о заседаниях
- собери события без галлюцинаций
- запиши `Хронология дела.md` вручную
- обнови `.vassal/case.yaml` и проверь YAML-валидность
- лог сохрани в `.vassal/codex-logs/{дата-время}-timeline-fallback.md`

## Идемпотентность

Повторный запуск безопасен:

- события `source:user` никогда не перезаписываются
- дубли не создаются
- при `extend` добавляются только новые события
- при `rebuild` автоматически сохраняются все ручные события юриста
