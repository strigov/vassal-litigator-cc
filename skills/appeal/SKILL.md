# appeal -- Апелляционная жалоба

Анализ решения суда первой инстанции, выявление оснований для обжалования, составление апелляционной жалобы в .docx через arbitrum-docx.

## Предварительные условия

- Дело инициализировано: `.vassal/case.yaml` существует.
- Решение суда первой инстанции получено (md-зеркало или файл для обработки).
- Желательно: полный контекст дела (позиции, отчеты о заседаниях, доказательства).

## Входные данные от Сюзерена

Перед началом уточни:

1. **Решение загружено?** (если нет -- запросить через add-opponent или прямую загрузку)
2. **В какой суд подаем?** (апелляционный суд определяется по первой инстанции, но уточнить)
3. **Какие основания видит Сюзерен?** (его мнение о том, что суд сделал неправильно)
4. **Срок подачи?** (по умолчанию: 1 месяц со дня изготовления решения в полном объеме -- ст. 259 АПК РФ, ст. 321 ГПК РФ; в особых случаях иные сроки)
5. **Есть ли особые указания?** (акценты, стратегия)

## Алгоритм

### Фаза 1 -- Загрузка контекста

1. Прочитай `.vassal/case.yaml` -- стороны, суд, предмет, хронология.
2. Прочитай `.vassal/index.yaml` -- реестр документов.
3. Загрузи решение суда первой инстанции (md-зеркало).
4. Загрузи все позиции из корня дела.
5. Загрузи отчеты о заседаниях из `.vassal/hearings/`.
6. Загрузи md-зеркала ключевых доказательств.
7. Загрузи наши процессуальные документы и документы оппонента.

### Фаза 2 -- Анализ решения (Opus-субагент)

Делегируй Opus-субагенту (model: "opus") глубокий анализ решения:

8. **Структурный разбор решения:**
   - Что установил суд (фактические обстоятельства)
   - На какие нормы сослался
   - Какие доводы сторон принял/отклонил
   - Мотивы отклонения наших доводов
   - Резолютивная часть (что решено)

9. **Поиск оснований для обжалования (ст. 270 АПК РФ / ст. 330 ГПК РФ):**

   **a) Неполное выяснение обстоятельств, имеющих значение для дела:**
   - Какие обстоятельства суд не исследовал
   - Какие доказательства не оценил
   - Какие доводы сторон проигнорировал

   **b) Недоказанность обстоятельств, которые суд счел установленными:**
   - Какие выводы суда не подтверждены доказательствами
   - Противоречия между выводами суда и материалами дела

   **c) Несоответствие выводов суда фактическим обстоятельствам:**
   - Логические противоречия в мотивировочной части
   - Неправильная оценка доказательств
   - Выводы, не следующие из установленных обстоятельств

   **d) Нарушение или неправильное применение норм материального права:**
   - Неприменение закона, подлежащего применению
   - Применение закона, не подлежащего применения
   - Неправильное истолкование закона

   **e) Нарушение норм процессуального права:**
   - Рассмотрение дела в отсутствие надлежаще не извещенного лица
   - Нарушение правил подсудности
   - Рассмотрение дела в незаконном составе
   - Нарушение тайны совещательной комнаты
   - Отсутствие протокола судебного заседания
   - Иные существенные нарушения

10. **Ранжирование оснований:**
    - По силе (от наиболее перспективного к наименее)
    - Безусловные основания для отмены (ч. 4 ст. 270 АПК РФ / ч. 4 ст. 330 ГПК РФ) -- приоритет

11. **Проверка сроков:**
    - Дата изготовления решения в полном объеме
    - Крайний срок подачи жалобы
    - Если срок пропущен -- основания для восстановления

### Фаза 3 -- Стресс-тест оснований

12. Для каждого основания:
    - Как оппонент будет возражать
    - Насколько устойчиво основание при проверке апелляцией
    - Есть ли судебная практика в нашу пользу

13. Покажи Сюзерену:
    - Список оснований (ранжированный)
    - Оценку перспективности каждого
    - Рекомендацию: какие основания включить в жалобу, какие отбросить
    - Предупреждение о сроках

14. Дождись подтверждения.

### Фаза 4 -- Генерация текста

15. Сформируй текст апелляционной жалобы. Структура:

**Шапка:**
- Наименование апелляционного суда
- Стороны (с процессуальным положением)
- Реквизиты обжалуемого решения (суд, дата, номер дела)

**Вводная часть:**
- Какое решение обжалуется
- Краткое изложение сути спора
- Чем заявитель не согласен (общая формулировка)

**Основания для отмены (по каждому основанию):**
- Что установил/не установил суд
- Почему это неправильно (со ссылками на доказательства)
- Норма права, которая нарушена
- Как это повлияло на правильность решения

**Просительная часть:**
- Решение отменить (полностью/в части)
- Принять новый судебный акт / направить на новое рассмотрение
- Распределение судебных расходов

**Приложения:**
- Копия обжалуемого решения
- Документ об уплате госпошлины
- Доказательства направления жалобы другим лицам
- Доверенность

16. Покажи текст Сюзерену. Итерируй по замечаниям.

### Фаза 4.5 -- Контрольное ревью (Codex xhigh, read-only)

17. Сохрани черновик жалобы + ранжирования в `.vassal/drafts/{ГГГГ-ММ-ДД}-appeal-draft.md`.
18. Прочитай `skills/codex-invocation/SKILL.md`.
19. Собери prompt из `prompts/analytical-reviewer.md`, подставь:
    - `output_path`: `.vassal/drafts/{ГГГГ-ММ-ДД}-appeal-draft.md`
    - `original_input`: `апелляционная жалоба: основания обжалования и ранжирование`
    - `case_root`, `plugin_root`, `extra_constraints`, `report_contract` — по стандарту
20. Проверь: `grep -c "{{"` → `0`.
21. Диспатч: `codex-companion.mjs task --background --effort xhigh` (БЕЗ `--write`).
22. Мониторь статус (until/case loop, sleep 25). Fetch result.
23. Разбор результата:
    - `REVIEW_OK` → переходи к фазе 5 (Оформление в .docx). `NITS` покажи Сюзерену как `ℹ️ FYI`.
    - `REVIEW_BLOCKING` → сохрани `.vassal/reviews/{ГГГГ-ММ-ДД}-appeal.md` с фронтматтером:
      ```yaml
      ---
      skill: appeal
      target_output: .vassal/drafts/{ГГГГ-ММ-ДД}-appeal-draft.md
      reviewer: codex-xhigh
      reviewed_at: <ISO datetime>
      verdict: REVIEW_BLOCKING
      blocking_count: <N>
      nits_count: <M>
      ---
      ```
      Покажи Сюзерену 3 опции:
      - `(a)` принять как есть
      - `(b)` один Opus фикс
      - `(c)` вручную
      Если выбран `(b)` — один раунд доработки по BLOCKING-блокам и затем повтор с фазы 4.5. Если выбран `(a)` — добавь `<!-- reviewed: accepted-over-objection -->` в черновик. Если выбран `(c)` — жди правки.

### Фаза 5 -- Оформление в .docx

24. Сохрани как md:
    ```
    {ГГГГ-ММ-ДД} Апелляционная жалоба.md
    ```

25. Спроси: "Оформить в .docx?"

26. Если да -- через arbitrum-docx:
    - **Тип 1** (процессуальный с шапкой)
    - Адресат: апелляционный суд
    - Имя файла: `ГГГГ-ММ-ДД Клиент Апелляционная жалоба.docx`

27. Помести в корень дела.

### Фаза 6 -- Обновление состояния

28. Обнови `.vassal/case.yaml`:
    - Добавь событие: подготовка апелляционной жалобы
    - Обнови статус дела (если применимо)
29. Обнови `.vassal/history.md`.
30. Обнови `index.yaml` (новый документ).

## Приоритеты в аргументации жалобы

1. Безусловные основания для отмены (ч. 4 ст. 270 АПК РФ) -- если есть, ставить первыми
2. Нарушение/неправильное применение норм материального права -- самые перспективные
3. Несоответствие выводов обстоятельствам -- конкретные противоречия
4. Процессуальные нарушения -- если существенные

## Граничные случаи

- Решение полностью в нашу пользу, но Сюзерен хочет обжаловать часть -> жалоба на часть решения
- Срок подачи пропущен -> подготовить ходатайство о восстановлении срока + жалобу
- Решение еще не изготовлено в полном объеме -> предупредить о сроке, начать анализ по резолютивной части
- Оппонент тоже подал апелляцию -> учесть при составлении, подготовить возражения на его жалобу

## Маршрутизация моделей

- **Opus**: анализ решения и отбор оснований обжалования.
- **Sonnet**: preview, сборка текста жалобы, сохранение и оформление.
- **Codex xhigh**: контрольное ревью (фаза 4.5) — read-only, аудируемый формат BLOCKING.
