# Catalog — Каталогизация документов дела

preview→apply: preview строит Claude-main (read-only), apply выполняет Codex medium (--write).

## Предварительные условия

- Существует `.vassal/index.yaml` с записями о документах
- Существуют md-зеркала в `.vassal/mirrors/`
- Если индекс пуст — сообщи Сюзерену, что сначала нужно загрузить документы через intake

## Фаза 1 — Preview (Claude-main, read-only)

1. Прочитай `.vassal/index.yaml`.
2. Определи документы с пустым или коротким `summary` (`< 20` символов). Составь список `doc-ID` для обогащения.
3. Покажи Сюзерену план:
   - `N` документов всего в индексе
   - `N` документов будут обогащены `summary` (список: `id` + `title`)
   - будет создан или перезаписан файл `Таблица документов.xlsx`

ЗАПРЕЩЕНО в этой фазе: любые записи в `index.yaml`, запуск скриптов, создание файлов.

## Фаза 2 — Подтверждение

Дождись ответа Сюзерена. Если список `doc-ID` или объём работы меняется — пересобери preview и покажи заново. Если операция отменена — остановись без изменений.

## Фаза 3 — Apply (Codex medium --write)

1. Прочитай `skills/codex-invocation/SKILL.md`.
2. Собери prompt из `prompts/file-executor-catalog.md`, подставь:
   - `case_root`
   - `plugin_root` по 3-tier fallback из `skills/codex-invocation/SKILL.md`
   - `plan_body`: список `doc-ID` для обогащения + подтверждение генерации `xlsx`
   - `extra_constraints`: пусто
   - `report_contract`: из `prompts/_preamble.md`
3. Проверь, что все плейсхолдеры раскрыты:
   `grep -c "{{" prompt.txt`
   Результат должен быть `0`.
4. Диспатч задачу:
   `codex-companion.mjs task --background --write --effort medium`
5. Мониторь выполнение по шаблону из `skills/codex-invocation/SKILL.md`.
6. Получи итог через fetch/result.

## Фаза 4 — Верификация (Claude-main)

1. Прочитай отчёт Codex.
2. Проверь: `Таблица документов.xlsx` создана, либо в отчёте явно зафиксирован CSV fallback; `.vassal/index.yaml` валиден, в отчёте нет изменений вне согласованного плана.
3. Сохрани лог в `.vassal/codex-logs/{ГГГГ-ММ-ДД-ЧЧмм}-catalog.md`:
   prompt + отчёт Codex.
4. Покажи Сюзерену резюме: сколько документов в индексе, сколько `summary` обновлено, путь к `Таблица документов.xlsx` или CSV fallback, если `openpyxl` отсутствовал.

## Fallback без Codex

Если Codex недоступен:

- Выполни apply-фазу самостоятельно: обогати `summary` через чтение зеркал и запусти
  `python3 [PLUGIN_ROOT]/scripts/generate_table.py --case-root "[CASE_ROOT]"`
- Лог сохрани в `.vassal/codex-logs/{дата-время}-catalog-fallback.md`.

## Идемпотентность

Повторный запуск `catalog` не создаёт дублей — он перезаписывает `Таблица документов.xlsx` и обновляет только пустые или неполные `summary` в `index.yaml`. Существующие достаточные описания не перезаписываются.

## Граничные случаи

- Документ без md-зеркала → включи его в preview как blocker для apply.
- Очень много документов (>100) → группируй список `doc-ID` в preview, но apply всё равно должен использовать точный перечень.
- `Таблица документов.xlsx` уже существует → предупреди в preview, что файл будет перезаписан.
